Juice

Juice is the simplest and most versatile dynamic JavaScript animation engine for the HTML Canvas (with jQuery Plugin support). Make animations that fly in seconds. You can even adjust them while they are actively animating.

Juice can save quite a bit of bandwidth transfer, and the processing is light. At < 7k minified, it can offer much more than any standard animated GIF image accent.

Getting Started

First, if you are using jQuery (necessary at the moment), make sure to include it. Then simply load the Juice library:


<script src="path/to/jquery.min.js">
<script src="path/to/juice.min.js">

Once included, Juice is made to allow you to get started as quickly as possible. This means that nearly anything may be omitted for defaults and you may make a looping animation with as little code as:


<script>
$('#my-id').Juice({
    points:[{
        paths:[{
            name:'arc'
        }]
    }]
}).play();
</script>

If '#my-id' is a Canvas, it will play right there. If not, Juice will create the Canvas for you. The selector doesn't even have to be a CSS ID. If you make it a class, it will copy the animation in all of the elements that match the selector. Don't forget to include play(), otherwise, the animation won't even appear. From there, you can simply add and change properties until you get what you want.

Important: Like any script acting on an HTML tag, the animation script must be placed below the tag it is acting on. Often, right above the </body> is best.

How it Works

Unlike other JS animators, Juice maximizes on the concept of relative time. Speed is determined by how far each object has to move and how much time it has to get there. This makes frames consistent at both high and low FPS. Motion Paths and Trailing Effects all utilize this relative time to determine how long it is in relation to the overall animation.

To improve performance, Juice also focuses all calculations on Motion Points. Rather than tracking 50 points on a trail, Juice tracks one and the rest are drawn relative to the first. So point distance vs. speed calculations are unnecessary, and, in fact, inhibit the process.

Another major draw to Juice is its cascading properties. If you set a point size on the animation, all points that don't have a size property will use it. You may override those properties as much or as little as you like. This allows every point to share or keep their own as needed.

Finally, by default, every animation is cached on its first run. This makes subsequent loops and plays bypass their calculations. However, you have complete control of the cache, even to the point that it may be turned off and on at will.

Documentation

Animations in Juice are entirely dynamic. At any point in time, you may change any single property of any object used by the animation. Additionally, Juice allows you to respond to specific events. These give access to the animation's properties, even allowing conditional behavior.
